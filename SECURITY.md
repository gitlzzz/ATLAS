# Security Policy

## Supported Versions

ATLAS is currently in active development under the `0.x` semantic versioning range. Security fixes are applied directly to the latest available release. 

| Version | Supported          |
| ------- | ------------------ |
| Latest Stable Release | :white_check_mark: |
| All Older Versions   | :x:                |


> **Note on Versioning:** This support policy applies during the initial development phase (`0.x`). Once ATLAS reaches its first major stable release (`1.0.0`), a formal Long-Term Support (LTS) schedule and version matrix will be established here.

## Reporting a Vulnerability

Please report security vulnerabilities through [GitHub Security Advisories](https://github.com/LopezGroup-ICIQ/ATLAS/security/advisories/new).

What to expect:
- You will receive a confirmation within 48 hours.
- We will evaluate the report and determine the impact, aiming to provide a detailed response within 7 days.
- If accepted, we will develop and coordinate a fix. You will be credited in the advisory (unless you prefer anonymity).
- If declined, we will provide a brief explanation.

For non-critical security concerns (e.g., dependency update suggestions), opening a regular issue or Pull Request is perfectly fine.
